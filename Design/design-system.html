10 — Principles
Motion

Motion serves navigation and feedback — never decoration. One orchestrated page-load reveal. All transitions use --ease-out for responsiveness.

Page load stagger

Hero content fades + slides up with staggered delay (60ms between elements). Logo → tagline → search bar → CTA chips.

animation-delay: 0, 60ms, 120ms, 180ms

Card hover

Cards lift 2px vertically + deepen shadow. No scale transforms — keeps layout stable on mobile.

translateY(-2px) · 220ms ease-out

Button press

Pressing a button drops 1px (translateY(1px)) to give tactile feedback. Releases on pointer-up.

:active translateY(1px) · 120ms

Filter chip toggle

Active state animates background + color fill over 120ms. No scale. Chip group scrolls horizontally on mobile with momentum.

background · color · 120ms ease-out

Search focus

Search bar grows shadow, border color transitions to Moss. The hero search also expands slightly (scale 1.01) to draw focus.

box-shadow · border-color · 220ms

Never use

Looping animations, auto-playing carousels, motion that obscures content, or transitions longer than 400ms on interactive elements.

Respect prefers-reduced-motion — all transitions → 0ms if set.

🌿 Guiding principle: Motion should feel like a gentle walk in the woods, not a sprint. If an animation makes the product feel busier, remove it.
